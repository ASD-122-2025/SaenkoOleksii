1.Оцінити асимптотичну складність алгоритму бінарного пошуку в О-нотації.¶

def bin_search(a_list, x, left, right):
if left > right or len(a_list) == 0:
return -1
middle = (left + right) // 2
if a_list[middle] == x:
return middle
elif (a_list[middle] < x):
return bin_search(a_list, x, middle + 1, right)
else: # a_list[middle] > x
return bin_search(a_list, x, left, middle - 1)

Асимптотична складність алгоритму бінарного пошуку в О-нотації складає O(log n), де 𝑛 n — кількість елементів у відсортованому масиві чи списку.

3.Написати функцію, яка б могла повертати датафрейм з таким складом полів: («n», «time»), де n – розмір масиву для функції bin_searh();¶

def bin_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
def create_dataframe(max_n, step):
    data = []
    for n in range(step, max_n + 1, step):
        arr = sorted(random.sample(range(1, 10 * n), n))
        target = random.choice(arr)
        start_time = time.time()
        bin_search(arr, target)
        end_time = time.time()
        data.append({"n": n, "time": end_time - start_time})
    df = pd.DataFrame(data)
    return df

5. Оцінити, який з двох алгоритмів є ефективнішим і в якому діапазоні розміру задачі.¶

Для малих масивів (до кількох тисяч елементів), лінійний пошук може бути швидшим, оскільки сортування для бінарного пошуку додає витрати. Для середніх та великих масивів, бінарний пошук є значно ефективнішим, якщо масив відсортовано або потребує лише одного сортування.

Контрольні питання¶

  • Сформулювати задачу пошуку. Нехай є масив цілих чисел A довжиною n. Потрібно знайти індекс елемента target у цьому масиві за допомогою двох алгоритмів пошуку: Лінійний пошук: Алгоритм, який перевіряє кожен елемент масиву по черзі, поки не знайде шукане значення target або не завершить перевірку всіх елементів. Бінарний пошук: Алгоритм, який працює тільки на відсортованих масивах. Спочатку масив сортується (якщо він не відсортований), після чого застосовується бінарний пошук для знаходження елемента target. Алгоритм використовує принцип поділу масиву навпіл, що дозволяє значно зменшити кількість перевірених елементів.
  • Які критерії можна використовувати для оцінювання ефективності алгоритмів пошуку? Часова складність, Просторова складність, Наявність попередньої обробки даних, Стійкість до змін у даних, Швидкість на різних типах вхідних даних, Простота реалізації, Адаптивність до зміни вимог, Швидкодія в умовах великої кількості запитів
  • Що таке лінійний пошук і яка його асимптотична складність? Лінійний пошук — це алгоритм пошуку елемента в масиві чи списку, при якому кожен елемент перевіряється по черзі, поки не буде знайдений шуканий елемент або не буде досягнутий кінець списку.
  • Бінарний пошук — це ефективний алгоритм пошуку елемента в відсортованому масиві, який використовує принцип поділу пополам для зменшення кількості перевірок. Його основна ідея полягає в тому, що замість перевірки кожного елемента масиву по черзі (як у лінійному пошуку), бінарний пошук дозволяє значно скоротити час виконання, зменшуючи розмір пошукової області вдвічі на кожному кроці.
  • Які переваги і недоліки використання бінарного пошуку порівняно з лінійним? Лінійний пошук ефективний для невеликих або не відсортованих масивів, тоді як бінарний пошук значно швидший на великих відсортованих масивах, але вимагає попереднього сортування.
  • Які ще алгоритми пошуку існують, окрім лінійного і бінарного? Hashing, BST, Алгоритми для пошуку в графах, Jump Search, Interpolation Search, Exponential Search, Ternary Search, K-d Tree, Fibonacci Search.
